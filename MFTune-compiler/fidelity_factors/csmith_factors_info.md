# Csmith (Factor Descriptions)

| Parameter                         | Description                                                                 | Enum Values                        | Cost Related | Notes                                                                                  |
|----------------------------------|-----------------------------------------------------------------------------|------------------------------------|--------------|----------------------------------------------------------------------------------------|
| `--inline-function-prob`         | Probability (%) that functions will be inlined.                            | 18, 62, 96, 97, 100                | Yes          | Higher values reduce function call overhead but may increase main function complexity. |
| `--max-expr-complexity`          | Maximum complexity of expressions.                                         | 1, 2, 3, 4, 5                      | Yes          | Affects nesting of operators, function calls, and pointer expressions.                |
| `--max-array-len-per-dim`        | Maximum array length per dimension.                                        | 34, 46, 67, 82, 100               | Yes          | Larger arrays increase memory access complexity and may impact runtime cost.          |
| `--max-block-size`               | Maximum number of statements per code block.                               | 1, 5, 6, 8, 10                    | Yes          | Influences control-flow complexity and opportunities for compiler optimizations.      |
| `--max-funcs`                    | Maximum number of functions to generate.                                   | 2, 6, 36, 63, 80                  | Yes          | Affects program size, inter-procedural analysis, and inlining opportunities.          |
| `--int128`                       | Enable 128-bit integer type `__int128`.                                    | true, false                       | Yes          | Tests compiler support for wide integers and impacts arithmetic complexity.           |
| `--unions`                       | Enable use of union types.                                                 | true, false                       | Yes          | Tests type aliasing behavior and potential undefined behavior.                        |
| `--structs`                      | Enable use of struct types.                                                | true, false                       | Yes          | Adds complexity in data structure layout and field access.                            |
| `--return-structs`               | Allow functions to return struct values.                                   | true, false                       | Yes          | Triggers specific calling conventions and return value handling.                      |
| `--uint8`                        | Enable use of 8-bit unsigned integer types (`uint8_t`).                    | true, false                       | Yes          | Tests small integer optimizations and potential register packing.                     |
| `--return-unions`                | Allow functions to return union values.                                    | true, false                       | Yes          | Similar to `--return-structs`, but with more type ambiguity.                          |
| `--no-signed-char-index`         | Prevent signed chars from being used as array indices.                     | true, false                       | Yes          | Helps avoid undefined behavior due to negative indices.                               |
| `--identify-wrappers`            | Generate wrapper functions around real implementations.                    | true, false                       | Yes          | Adds indirection in function calls, useful for testing call graphs.                   |
| `--no-hash-value-printf`         | Disable the final hash-based output of the generated program.              | true, false                       | Yes          | Reduces I/O overhead slightly; useful when only code generation is needed.            |
| `--bitfields`                    | Enable generation of bitfield members in structs.                          | true, false                       | Yes          | Useful for testing memory layout and alignment-sensitive optimizations.               |
| `--fresh-array-ctrl-var-names`  | Use unique names for array control variables to avoid name conflicts.      | true, false                       | Yes          | Slightly increases readability and avoids scope conflicts in generated code.          |
